​

​ 使用 winhex 等工具查看 php_xmlrpc.dll，可以发现在文件中存在危险字符 eval

2. 定位危险函数

​ 首先是要确定恶意代码的位置。IDA x64打开该dll后，通过查找字符串列表，接着筛选出eval字符（注：eval() 函数把字符串按照 PHP 代码来执行）就可找到实际后门代码位置。

​ 双击点击，查看详细位置及内容

​ 交叉引用，找到具体代码点

​ 按F5生成伪代码，如图

​ 通过上述方法共可以找到两处利用点

3. 还原压缩数据

​ 可以看出上述代码执行了会将1000D028-1000D66C区间内(偏移地址是D039-D66C)的按字节(将Dword处理成char)获取，使用PE结构分析工具可以看到D039-D66C是出于.data球衣退役的

​ 那么我们需要详细查看在 hex编码中1000D028-1000D66C这一段的内容具体是什么，可以看见在hex数据中有gzuncompress表示字符，怀疑这一段数据是通过gzuncompress压缩的

​ 同理查看另一处eval利用点，它会尝试将1000D66C-1000E5C4(偏移地址是D66C-E5C4)这一段的内容获取。尝试使用gzuncompress的解压缩脚本获取源代码

​ 我们需要提取这两段压缩过后的数据解压再进行后续分析，原有的[轮子][http://mp.weixin.qq.com/s?__biz=MzI5NjA0NjI5MQ==&mid=2650165920&idx=1&sn=ac45922b6cf1db0f3d3cf0a10872be06&chksm=f448a91cc33f200a32cdbd01535e227a4a81cd3ce843992e410d0e4d5b772914d1ac3d6324fe&mpshare=1&scene=1&srcid=&sharer_sharetime=1569082336079&sharer_shareid=050fef71c2c8c2cd7ebc8d5cccf6b556#rd]是微步在线分析的，但是在我本地python环境下运行不了，所以我修改里里面的部分内容

​ 运行后会生成两段解压后的数据，不过此时的数据已经base64编码过。我们对他们分别进行解码。

4. 构造利用链

​ 恶意代码处于sub_100031F0函数中

​

​ 在上面发现的两段可利用内容的基础上往上分析，可以发现spprintf(&v40, 0, "%s;@eval(%s('%s'));", v34, aGzuncompress, v40);该代码如果要被执行，首先if ( !v12 )的条件需要满足，但是v12 = strcmp(**v32, aCompressGzip);

​ 说明对该变量有硬编码字符串的比较，需要比较v12与compress,gzip是否一致

​ 在往上可以看见一个else说明当前是在if语句块中，那么尝试查看一下语句块中的内容

​ 这里需要对$_SERVER['HTTP_ACCEPT_ENCODING'] 这个参数进行判断，判断请求内容中是否存在HTTP_ACCEPT_ENCODING，且请求数据是否为gzip,deflate，如果是才能进入内部的if判断从而导致接下来的命令执行触发可能性出现。

​ 再看看里面的那个条件`if ( zend_hash_find(*(_DWORD *)(*a3 + 4 * executor_globals_id - 4) + 216, aServer, strlen(aServer) + 1, &v37) != -1 && zend_hash_find(**v37, aHttpAcceptChar, strlen(aHttpAcceptChar) + 1, &v35) != -1 )，

​ 这里尝试对$_SERVER['HTTP_ACCEPT_CHARSET'] 这个参数进行判断，判断请求内容中是否存在HTTP_ACCEPT_CHARSET，例如：“iso-8859-1,*,utf-8”。如果也存在的话就接着取HTTP_ACCEPT_CHARSET字段值，对该值进行base64解码

​ 对 v35 进行 Base64 解码，并将解码后的结果传递给 zend_eval_string 函数进行后续操作。通过调用zend_eval_string(v38, 0, &byte_10012884, a3);但是我们需要让后门代码执行。以上是真的情况，如果上面的判断结果为假，则直接跳过，来到v12 = strcmp(**v32, aCompressGzip);对其判断，如果字符比较相等就继续执行下面的unk_1000D66C-unk_1000E5C4(偏移D66C-E5C4)这段内容调用spprintf(&v40, 0, "%s;@eval(%s('%s'));", v34, aGzuncompress, v40);
